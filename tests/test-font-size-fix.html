字号转换修复测试

问题说明

在导出DOCX文档时，web中设置的字号与Word中显示的字号不一致。

原因：px到pt的转换系数不正确。之前使用1.33，正确应该是0.75。

转换对比测试

16px 文本样例（旧转换）
旧转换：16px × 1.33 × 2 = 42.56 ≈ 43 半点 = 21.5pt
16px 文本样例（新转换）
新转换：16px × 0.75 × 2 = 24 半点 = 12pt
20px 文本样例（旧转换）
旧转换：20px × 1.33 × 2 = 53.2 ≈ 53 半点 = 26.5pt
20px 文本样例（新转换）
新转换：20px × 0.75 × 2 = 30 半点 = 15pt
24px 文本样例（旧转换）
旧转换：24px × 1.33 × 2 = 63.84 ≈ 64 半点 = 32pt
24px 文本样例（新转换）
新转换：24px × 0.75 × 2 = 36 半点 = 18pt

标准转换公式

  • CSS像素到点：1px = 0.75pt
  • Word半点单位：1pt = 2半点
  • 完整转换：px × 0.75 × 2 = 半点数

修复内容

  1. 修复了 WordHandler.ts 中的 convertFontSize 函数
  2. 修复了 test-export.html 中的 convertFontSize 函数
  3. 将转换系数从 1.33 改为 0.75
  4. 添加了详细的注释说明转换逻辑

验证方法

1. 在编辑器中设置不同字号的文本

2. 导出为DOCX文档

3. 在Word中打开，检查字号是否与预期一致

4. 预期结果：16px应显示为12pt，20px应显示为15pt，24px应显示为18pt
